ETL工具之Kettle

一、 Kettle是什么

Kettle是纯Java编写的ETL（Extract-Transform-Load）数据抽取-转换-装在工具。现在改名叫为PDI（数据集成）

Kettle的图形界面为Spoon

对于Kettle的使用就是 tra转换 和 job作业。job中包含transformations

  • 转换：
    • 对数据的ETL工作
    • 是对数据进行处理，一个转换中的各个组件是一同开启的。
  • 作业：是对整个工作流的控制， 一步一步的。

二、Kettle安装、DB连接

Kettle必须需要JDK，所以本地环境的JAVA_HOME必须注意jdk版本。

如8.2版本的Kettle，需要使用jdk8, 亲测jdk11会运行异常，只要进入spoon界面，对组件进行右键就会崩溃卡死

DB连接：需要将驱动包放置Kettle的lib目录下，需要下载对于版本的db驱动包 。但是db驱动包的版本需要注意，应该是Kettle8.2不支持高版本的如mysql8.x的驱动包(亲测会无法连接).，所以如果需要连接mysql8.x版本，需要mysql5.1.48的驱动包，因为其兼容5.X和8.X

三、转换

对于Kettle的学习主要就是在于学会使用转换的各个组件。

转换就是Kettle对于数据ETL的全部工作

转换由步骤组成，步骤之间使用跳进行连接。

转换的多个控件

输入控件

输入控件是完成ETL中Extract,即对数据的抽取。

在确认之前，往往都要先获取字段。

常用的有：csv文件输入、Excel文件输入、表输入、JSON、XML等。

输入控件需要注意的就是获取字段时，需要正确的格式，否则后续会需要自己更改，如文本文件输入中的字段：Number类型的默认格式为0.00，需要更改为0，或者更改类型为Integer.

输出控件

输出控件是完成ETL中的Load，即对数据的加载。

常用的有：Excel输出、Sql文件输出、表输出、更新、插入/更新、删除、数据同步

更新、插入/更新、删除、数据同步区别：

  • 更新:只能更新，所以需要要求两表行数一直，否则报错

  • 插入/更新:两表比较后可以插入和更新，但是与数据同步相比，无法删除。不能完全同步。

    • iglbk.l，吧fflagfield字段四个值的含义

      • identical-相同

转换控件

转换控件主要用来完成ETL中的Transform数据转换工作。T的工作耗时一般占用ETL耗时的2/3。常有的控件有：Concat fields、值映射、增加常量/序列、计算器、字符串剪切/替换/操作、排序记录、去除重复记录、唯一行、拆分字段、列拆分为多行、行扁平化、列转行、行转列

  • Concat fields:多个字段组合为一个字段。

  • 值映射：当一个字段的值匹配时，可以映射成其他值

  • 增加常量、序列：两者都可为元数据增加字段

    • 增加常量增加一列统一的常量

    • 增加序列可以指定增长的步长

  • 计算器：根据指定的计算规则，新生成一列字段存储结果

  • **字符串~**：对字符串操作

  • 排序记录：根据指定的字段进行排序

    • 排序记录通常配合其他控件使用，eg:去除重复记录、合并记录等

  • 去除重复记录&唯一行：通过指定字段对比，去除重复记录

    • 唯一行是对每一行的数据建立hashcode，不需要排序，速度也更快。
    • 去除重复记录：是通过排序后，每两行进行对比

  • 拆分字段：将一个字段拆分为多个字段，需要注意分隔符

    ​

  • 列拆分多行:行拆分为多列

  • 行扁平化：列拆分多行的逆向操作，需要注意的是

    • 需要排序
    • 每组记录条数需要相同，否则数据会错乱

  • 列转行、行转列

    • 列转行：将一列的值转为行，需先分组排序

应用控件

应用控件可以看作是kettle给我们提供的工具类。

常用的有：替换null值（将null值替换为其他值）、写日志（用于调试）

流程控件

流程控件用于控制数据流程和数据流向

常用的有：Switch/case、过滤记录（相当于if）、空操作（流程终点）、中止

  • 中止：如果数据流留下此处，转换就会中止。中止的三个选项

    Option Description
    Abort the running transformation Select to stop the transformation when the Abort threshold is reached.
    Abort and log as an error Select to ==log== an error report when the Abort step stops the transformation. This option is useful when a parent step (such as the Kafka Consumer step) executes a sub-transformation containing the Abort step, where the Abort step will cause the parent step to stop.
    Stop input processing Select to stop the input steps to the transformation when the Abort threshold is reached, while allowing any records already retrieved or initiated to be processed. 不中断其他的流
    Abort threshold阈值 Specify the number of rows at which to abort the transformation after an error is detected. For example, if the threshold is set to 0, the Abort step will stop the transformation after the first row is processed. If the threshold is set to 5, then the Abort step will stop the transformation after the sixth row is processed. The default value is 0.
    总结 前两个会中止其他流程，eg:过滤操作true操作，false中止，将会中止流向true的操作，但是由于是并发，不能马上中止时，也会有一些成功操作了。
    第二个人会使用日志来记录
    第三个不会中止

查询控件

查询控件用来查询数据流中的数据

常用：数据库查询（数据库和流中数据进行左连接，没啥用，没有记录集好用）、流查询（进行等值匹配）

连接控件

连接控件通常用来将多个数据集通过比对，合并成一个数据集

常用：合并记录、排序合并、记录关联/记录集连接

  • 合并记录：两个数据集的对比

    • 需要经过排序的两个数据源，一个新数据源、一个旧数据源。

    • 通过比对后，changefiled字段会显示对应的值。identical、new、changed、deleted。

  • 排序合并：多个经过排序的数据集合并成一个数据集

  • 记录关联：两个数据集进行笛卡尔积

  • 记录集关联: 两个数据集可以选择进行，左外、右外、内连接、全连接。两个步骤需要经过排序记录组件

映射控件

映射控件可以理解为存储过程，即就是定义一个子转换，优点为代码封装和重用

  • 映射输入规范：定义变量
  • 映射输出规范：啥也不做，表示映射结束
  • 映射：映射输入规范+操作+映射输出规范 = 映射

四、作业

作业是由作业项组成，作业项由作业跳连接

作业跳有三种

  1. 无条件执行
  2. 为真时执行
  3. 为假时执行

作业时串行的，也就是下一个作业项需要等到上一个作业项完成，才能得到结果对象。作业就是来控制并行的转换
